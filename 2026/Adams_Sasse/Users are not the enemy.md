## APA Citation:** Adams, A., & Sasse, M. A. (1999). Users are not the enemy. Communications of the ACM, 42(12), 40–46. https://doi.org/10.1145/322796.322806

## Core 5 Insights

### 1. Users are not inherently careless
The assumption that users are naturally insecure or irresponsible should be challenged.  
Security failures are not simply the result of user negligence.

### 2. Technical security without usability fails
Password research has focused on technical mechanisms, while usability has rarely been investigated.  
Ignoring usability leads to ineffective security in practice.

### 3. Multiple passwords reduce security
Requiring users to remember multiple passwords decreases memorability and increases insecure work practices (e.g., writing them down or simplifying them).

### 4. Incompatibility with work practices causes circumvention
If password mechanisms are incompatible with actual work practices, users perceive them as “not sensible” and circumvent them.

### 5. Lack of communication lowers security motivation
Insufficient communication between security departments and users leads to misunderstanding, low awareness of threats, and reduced motivation to comply.

## My Reflection

Although this paper was written in 1999, it accurately identified the limitations of technology-centered security. It emphasizes that security cannot be achieved merely by strengthening technical controls, and that technical security mechanisms which ignore user behavior are bound to fail.

This insight can be connected to the contemporary 2025 security trend of Zero Trust. While Zero Trust aims to eliminate implicit trust through technical enforcement, implementing a completely trustless structure purely through technology has practical limitations. Ultimately, effective security requires the integration of technical controls and an understanding of user behavior—an implication that this paper had already anticipated.

The core arguments of the paper highlight several structural issues: the oversimplified assumption that user carelessness is the root cause of security failures, the technical limitations of password mechanisms, the way overly complex security policies distort user behavior, and the structural causes behind insufficient security awareness. The authors critically examined the security paradigm of their time and revealed its inherent weaknesses.

Even in 2025, many of these issues remain unresolved and continue to be areas of active research. Analyzing human behavioral patterns and incorporating them into security design extends beyond pure technical domains; it requires interdisciplinary integration across sociology, behavioral psychology, and organizational studies.

In conclusion, I believe that the most viable solution lies in user-centered design, transparent sharing of threat information, policies that realistically reflect actual workflow, and security structures that minimize cognitive burden.

Personally, I have recently been studying sociology, and I am working toward developing a technological approach that integrates social structures and cybersecurity.
